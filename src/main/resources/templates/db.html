Реляционные базы данных (БД)

БД DB, Database — это организованная совокупность данных, предназначенная для их хранения и обработки.
СУБД DBMS, Database Management System — ПО, позволяющее создавать, изменять, поддерживать БД и управлять доступом к ней.
Модель данных Data model — описание способа представления и обработки данных.

БД в реляционной модели — совокупность отношений и связей между ними.
Отношение Relation, Table представляется двумерной таблицей.

Существуют и другие модели данных:

 сетевая, иерархическая, объектная и пр.

У отношений есть ряд ключевых свойств:

  • Каждый элемент — один элемент данных;
  • Все атрибуты однородны (одного типа данных);
  • Атрибуты однозначно именуются;
  • Все строки являются уникальными (не всегда соблюдается).

Сущность Entity — это тип объекта, информация о котором хранится в БД.
Кортеж Tuple, Record (запись) — набор пар атрибут-значение, представляется строкой.
Атрибут Attribute, Field (поле) — свойство сущности, представляется столбцом.
Домен Data domain (тип данных) — всё множество допустимых значений атрибута.
