Объектно-ориентированное программирование (ООП)

ООП Object-oriented programming — это парадигма, описывающая программу как взаимодействующие объекты.
ООП это также проектирование и объектное мышление.
В ООП важно выбирать правильный уровень абстракций.

Основные понятия:

  • Класс Class — описание и реализация абстрактного типа данных, шаблон объекта.
  • Объект Object — сущность с полями и методами, экземпляр класса
  • Интерфейс Interface — перечень объявлений методов, описание взаимодействия с классом.
  • Метод Method — процедура принадлежащая классу, способная обрабатывать его поля.
  • Поле Field — переменная, принадлежащая классу.

Основные принципы:

  • Абстракция Abstraction — это выделение значимых характеристик объекта, упуская незначимые.
  • Наследование Inheritance — механизм, для описания новых классов на основе существующих.
  • Инкапсуляция Incapsulation – объединение данных и методов их обработки; сокрытие реализации.
  • Полиморфизм Polymorphism — способность объектов с одинаковой спецификацией иметь различную реализацию.

Плюсы

  • Повышается степень повторного использования кода;
  • Как следствие можно создавать программы легче и быстрее;
  • При грамотном проектировании код легче понимать и поддерживать;
  • Способствует параллельной разработке, тестированию;
  • Сокращение числа межмодульных вызвовов;
  • Принципы ООП по сути и есть его достоинства.

Минусы

  • Снижение ресурсной эффективности накладные расходы;
  • Проектирование занимает больше времени и требует подготовки.
  • Программы быстро становятся сложными.
  • Сложно решать некоторые типы задач.
Откуда всё это?

Есть мнение, что ООП более естественно и является отражением речи, реального мира.
Объект ООП — существительное, объект реального мира с определенным уровнем абстракции.
Его описание, прилагательное (железный, большой, желтый, состоящий из звеньев...) — это свойства.
Доступные действия с ним, глаголы (включить, сложить, взаимодействовать...) — интерфейс.
Как изменить объект, для применения в других условиях или целях — наследование и полиморфизм.
Простое использование объекта без акцента на внутренее устройство — инкапсуляция.
